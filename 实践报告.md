# 编译原理实践报告

## 项目概述

本项目实现了 ToyC 语言到 RISC-V32 汇编的编译器。编译器使用 Java 编写，通过 Maven 构建，入口类为 `toyc.Main`。程序从标准输入读取 ToyC 源代码，向标准输出生成 RISC-V32 汇编代码。

## 编译器结构

编译器主体位于 `src/main/java/toyc`，核心模块包括：

- 词法分析器 `Lexer`：识别关键字、标识符、整数、运算符、分隔符，并跳过空白与 C 风格注释。
- 递归下降语法分析器 `Parser`：按照 ToyC 文法构造抽象语法树。
- 抽象语法树 `Ast`：使用 Java sealed interface 表达顶层定义、语句和表达式。
- 符号表 `Scope`：支持嵌套作用域、全局变量、局部变量、常量和函数符号查询。
- 代码生成器 `CodeGen`：遍历 AST 生成 RISC-V32 汇编。

源码按编译器阶段拆分为多个 Java 文件，`Main` 仅保留命令行入口，避免词法、语法、语义和后端逻辑混杂在同一个文件中。

## 词法与语法分析

词法分析阶段将输入源程序转换为 token 序列，支持：

- `const`、`int`、`void`、`if`、`else`、`while`、`break`、`continue`、`return` 等关键字。
- C 风格标识符。
- 十进制整数。
- 算术、关系、逻辑和赋值运算符。
- 单行注释与多行注释。

语法分析采用手写递归下降实现，按优先级拆分表达式解析函数：

- `parseOr`
- `parseAnd`
- `parseRel`
- `parseAdd`
- `parseMul`
- `parseUnary`
- `parsePrimary`

这种结构直接对应 ToyC 运算符优先级，便于处理左结合表达式。

## 语义处理

当前实现针对评测输入满足语义约束的前提，完成代码生成所需的语义信息维护：

- 全局作用域记录全局变量、全局常量和函数。
- 函数作用域记录形参。
- 语句块创建新的嵌套作用域。
- 局部变量按栈帧偏移分配存储位置。
- 常量声明在编译期求值，常量引用在生成阶段直接展开为立即数。
- 禁止对常量赋值。

常量表达式支持算术、关系、相等性和逻辑运算，并遵循逻辑与、逻辑或的短路求值规则。

## RISC-V32 代码生成

生成代码使用保守的栈式策略：

- 每个函数建立独立栈帧，保存 `ra` 和 `s0`。
- 局部变量和形参保存在当前函数栈帧中。
- 表达式结果统一放在 `a0`。
- 二元表达式用栈临时保存左操作数。
- 函数调用按 RISC-V 调用约定传参，前 8 个参数使用 `a0-a7`，其余参数放在调用者栈上。
- 调用前保持栈 16 字节对齐，支持嵌套函数调用。
- `if`、`while`、`break`、`continue` 和短路逻辑通过标签与条件跳转实现。

全局变量输出到 `.data` 段，函数输出到 `.text` 段，并导出 `main`。

## 构建与运行

构建命令：

```bash
mvn -q -DskipTests package
```

运行命令：

```bash
java -jar target/toyc-compiler-1.0.0.jar < input.tc > output.s
```

编译器接受 `-opt` 参数但当前不会启用额外优化。

## 测试

仓库中提供了基础回归脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\run-samples.ps1
```

脚本会构建项目，并测试以下场景能成功生成汇编：

- 基础 `main` 返回。
- 全局变量、常量、循环、条件分支和函数调用。
- 嵌套函数调用和超过 8 个参数的调用。
- `-2147483648` 整数边界。

由于当前开发环境没有 RISC-V 交叉编译器和 qemu，尚未在本机执行目标汇编验证退出码。

## 后续优化方向

- 增加面向非法输入的完整语义错误检查。
- 实现常量折叠、死代码删除等中端优化。
- 引入简单寄存器分配，减少表达式求值过程中的栈访问。
- 在具备 RISC-V 工具链的环境中加入端到端运行测试。
